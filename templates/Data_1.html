Data Modelling ERD to ETL process

Entity Relationship Diagram was created to segregate the data into four separate tables for data storage into PostgresSQL. Primary and Foreign keys were created accordingly to avoid data duplication. One Postgres SQL query file was created to complete the process of data loading for the user. The data sets contain three levels of classification: Property type, Suburb and Local Government Area. The datasets will be used for prediction of Victorian property prices. The data was extracted through api calls from Domain.com.au
